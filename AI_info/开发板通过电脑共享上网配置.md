# 开发板通过电脑共享上网配置

## 背景

- 电脑通过 WLAN 上网：IP 172.28.82.182，网关 172.28.80.1，DNS 后缀 bitmain-inc.com
- 板子（Sophon/linaro）通过网线直连电脑以太网口
- 目标：让板子通过电脑的网络访问互联网

## 方案：Windows ICS（Internet Connection Sharing）

### 步骤 1：Windows 端开启 ICS

1. 打开"网络连接"（`ncpa.cpl`）
2. 右键 **WLAN** 适配器 → 属性 → **共享** 选项卡
3. 勾选"允许其他网络用户通过此计算机的 Internet 连接来连接"
4. 家庭网络连接选择 **以太网**
5. 确定

开启后，Windows 会自动将以太网口 IP 设为 **192.168.137.1**，并在该接口启动 DHCP 服务器。

### 步骤 2：配置板子

#### 方法 A：DHCP（推荐）

板子设为 DHCP 自动获取，会自动拿到 192.168.137.x 的地址。

#### 方法 B：静态 IP

| 项目 | 值 |
|------|------|
| IP 地址 | `192.168.137.2` |
| 子网掩码 | `255.255.255.0` |
| 默认网关 | `192.168.137.1` |
| DNS | `8.8.8.8` 或 `114.114.114.114` |

板子上执行：

```bash
ifconfig eth0 192.168.137.2 netmask 255.255.255.0 up
route add default gw 192.168.137.1
echo "nameserver 8.8.8.8" | sudo tee /etc/resolv.conf
```

注意：`sudo echo "xxx" > /etc/resolv.conf` 会报 Permission denied，因为重定向由当前 shell 执行，sudo 只提升了 echo 的权限。需要用 `echo "xxx" | sudo tee /etc/resolv.conf` 代替。

### 步骤 3：验证

```bash
ping 192.168.137.1   # 测试能否到达电脑（网关）
ping 8.8.8.8         # 测试能否到达外网
ping baidu.com       # 测试 DNS 解析
```

## 遇到的问题及排查

### 问题 1：以太网显示"未识别的网络"

**正常现象**，不影响使用。直连板子没有路由器/DHCP 响应，Windows 无法识别网络类型。

注意：未识别的网络会被归类为公用网络，防火墙更严格。如果 ping 不通可临时关闭防火墙测试。

### 问题 2：DNS 解析失败（Temporary failure in name resolution）

`ping 8.8.8.8` 如果也不通，说明 ICS 转发未生效。

### 问题 3：ping 网关通但 ping 外网不通（ICS 转发未生效）

#### 解决方法 1：重新开启 ICS

1. 右键 WLAN → 属性 → 共享 → 先取消勾选 → 确定
2. 再次右键 WLAN → 共享 → 重新勾选 → 家庭网络选"以太网" → 确定

#### 解决方法 2：手动开启转发（管理员 PowerShell）

```powershell
Set-NetIPInterface -InterfaceAlias "WLAN" -Forwarding Enabled
Set-NetIPInterface -InterfaceAlias "以太网" -Forwarding Enabled
New-NetNat -Name "BoardShare" -InternalIPInterfaceAddressPrefix "192.168.137.0/24"
```

#### 解决方法 3：临时关闭防火墙排除干扰

```powershell
Set-NetFirewallProfile -Profile Domain,Public,Private -Enabled False
```
